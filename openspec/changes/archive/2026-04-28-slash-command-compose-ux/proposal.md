## 为什么要做这个改动

当前 Web 端键入斜杠命令后会立即执行，无法追加参数或补充上下文，且发送后的消息气泡展示的是命令背后的完整 Markdown 提示词，严重干扰对话可读性。

## 改动内容

- 选择斜杠命令后，在输入框上方显示一个可删除的「命令标签（Chip）」，不直接发送
- 用户可在 Chip 后方继续输入补充文本，再手动点击发送
- Chip 可通过点击 `×` 按钮或在输入框内容为空时按 Backspace 键删除
- 发送后，对话中的用户消息气泡**只显示用户追加的文本**（若无追加文本则显示命令名，如 `/commit`），不展示命令背后的完整 Prompt Markdown

## 涉及能力模块

### 新增能力模块

- `slash-command-token`：输入框内的斜杠命令 Chip/Token UI —— 将选中的命令渲染为可删除标签，支持追加输入，并控制已发送消息的展示内容
